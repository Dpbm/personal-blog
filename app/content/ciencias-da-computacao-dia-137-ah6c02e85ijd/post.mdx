-   abstração em que classes com relacionamento 1:N, você desmembra a o
    item em outra classe (Detail)








### Levantamento de requisitos

-   arte de extrair os problemas dos stakeholders
-   uma boa maneira de levantar requisitos é entender como o sistema é
    hoje








### Lista de requisitos

-   a linguagem deve ser não técnica, para todos os stakeholders
    entenderem








### Brown paper --- White paper

-   técnica antiga
-   maneira mais fácil de vender um sistema
-   coloca o sistema atual em um papel marrom e o novo em um branco,
    assim o cliente verá o antigo como se fosse algo velho e ruim








### Lista de serviços

-   apresenta as atividades que são feitas
-   não precisa ser só em software








### Mini especificação

-   breve descrição de um serviço e/ou objeto
-   bom para pegar regras de negócio








### Casos de uso completo

-   ao usar o "-" nas extensões, estamos dizendo que tal extensão cabe
    de x até y (x-y), além disso também podem ser adicionadas letras
    para vários casos, 1--10a, 1--10b, etc.
