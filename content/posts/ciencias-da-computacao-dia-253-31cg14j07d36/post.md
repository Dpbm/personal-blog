#### Imperativo

-   mais antigo
-   mais próximo de assembly
-   segue a sequência do código
-   lista de instruções que modificam valores na memória até atingir o
    estado final
-   sem loops, ou similares, apenas estados sendo modificados e GOTOs
    sendo realizado quando uma certa condição é verdadeira
-   baseado na arquitetura de Jon Von Neumann
-   não possui blocos, aqui, para realizar "sub-operações", é necessário
    usar GOTO, para ir para certas regiões do código








#### Turing complete

-   quando uma linguagem possui todas as peças necessárias para executar
    qualquer coisa que seja computacionalmente computável








#### Abstração de células de memória

-   simples → variáveis
-   sofisticadas → matrix, arrays, etc.
